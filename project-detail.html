Authentication & Shift Management Portal

Java SQL

The Problem

Organizations often struggle with managing user access securely while scheduling shifts. Existing tools were either too expensive or lacked specific security controls needed for role-based access.

The Solution

I built a secure portal that handles user registration, login, and shift allocation. The system uses role-based access control (RBAC) to ensure employees can only view their own data, while admins have full oversight.

Key Features

  • Secure Password Hashing (SHA-256)
  • Role-Based Access Control (Admin vs. User)
  • Automated Shift Conflict Detection
  • SQL Injection Prevention

What I Learned

This project taught me the importance of input validation in preventing security vulnerabilities. I also deepened my understanding of database normalization when designing the user/shift tables.
